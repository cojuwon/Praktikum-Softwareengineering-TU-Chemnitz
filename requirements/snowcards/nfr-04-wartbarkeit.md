# Requirement #: NFR-04

# Requirement Type: Non-functional

# Event/BUC/PUC #:

# Description:
- Das System benötigt eine saubere und dokumentierte Codestruktur (Kommentare, modulare Architektur, Namenskonventionen), die externe Wartung ermöglicht. 
- Die Änderungen am System sind ohne tiefgreifende Eingriffe möglich (modularer Aufbau).
- Einheitliche Namenskonventionen, Kommentare und nachvollziehbare Verzeichnisstrukturen sind gegeben.

# Rationale:
- Eine gute Wartbarkeit reduziert langfristig den Aufwand für Pflege, Anpassung und Fehlerbehebung.  
- Da die Verantwortung für den Betrieb und die Weiterentwicklung an Dritte übergeben wird, muss der Code ohne persönliche Vorkenntnisse wartbar sein.

# Originator:
- Projektteam und zukünftige (externe) Systembetreuer:innen.

# Fit Criterion:
- Code ist modular aufgebaut (Funktionalitäten sind klar getrennt).
- Mindestens 80 % der Hauptfunktionen und Module sind kommentiert.
- Jede Datei und Komponente folgt einer einheitlichen Benennungs- und Ordnerstruktur.
- Änderungen an einer Komponente erfordern keine Anpassungen in mehr als zwei weiteren Modulen.
